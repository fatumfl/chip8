# Знакомство с системой контроля версий git

1. Проверить в консоли установленную версию git:
git --version

2. Создать или перейти в папку, в которой будет располагаться наш репозиторий:
mkdir chip8
cd /chip8

3. Инициализировать новый репозиторий в целевой папке:
git init

4. Настроить параметры пользователя:
git config user.name "Имя.Фамилия"
git config user.email "sample@mail.com"
git config --local --list

5. Проверьте содержимое папки:
ls -a

6. Проверьте статус репозитория:
git status

7. Создайте следующую структуру папок и файлов в репозитории:

chip8/
	docs/
	roms/
	source/
	README.md

8. Добавьте все папки и файлы в список отслеживаемых:
git status
git add README.md		- добавит только указанный файл
git add .			- добавит все новые файлы в репозитории
git status

9. Зафиксируйте изменения в истории:
git commit -m "Comment"	- всегда используйте английский для комментариев с краткой сутью сделанных изменений
git status

10. Внесите изменения в файл README.md
	- На первой строке укажите имя проекта - CHIP-8 Emulator со стилем заголовка первого уровня
	- Укажите Имя, Фамилию, номер группы курсивом
	- Оставьте одну пустую строку и добавьте линию-разделитель для абзацев
	- Добавьте абзац с Lorem Ipsum

11. Зафиксируйте изменения:
git status
git add .
git status
git commit -m "Comment"

12. Просмотрите историю коммитов:
git log

13. Скачайте свежую версию TiddlyWiki и поместите в папку docs/ под именем index.html

14. Зафиксируйте изменения

# Настройка и публикация TiddlyWiki через систему GitHub Pages

24. Перейдите в Settings репозитория, группа Code and Automation, секция Pages.

25. Задайте следующие параметры:
	- Источник: Deploy from a branch
	- Branch: main	/docs
Нажмите Save

26. Перейдите по ссылке или нажмите Visit site вверху страницы. Если все сделано верно, то должна открыться страница TiddlyWiki.

27. В GitHub перейдите в настройки профиля Settings -> Developer Settings -> Personal Access Token -> Fine-Grained Tokens. Нажмите Generate new token. Задайте параметры:
	- Token name: TiddlyWiki
	- Description: TiddlyWiki Save Feature Integration
	- Expiration: No expiration
	- Repository access: Only select repositories - chip8
	- Permissions: Add permission -> Contents -> Read and write
	
28. Скопируйте токен. Открыть его второй раз - нельзя. Придется пересоздать, если не скопировали.

29. Откройте страницу TiddlyWiki в GitHub Pages. Перейдите в Control Panel -> Saving -> GitHub Saver. Задайте следующие параметры:
	- Username: Логин
	- Password: Скопированный токен
	- Target repository: логин/chip8
	- Target branch: main
	- Path to target file: /docs
	- Filename: index.html

30. Нажмите Сохранить (красную кнопку) в TiddlyWiki. Если все сделали верно, система сообщит об успешном сохранении.

31. Для синхронизации локального репозитория с удаленным после изменений в TiddlyWiki вводите команды:
git add .

# Автоматизация компиляции кода на C

35. Создайте файл main.c в папке source/

36. Напишите программу, которая выводит Hello, World! в консоль при запуске.

37. Синхронизируйте локальный репозиторий с удаленным.

38. Создайте файл и папки по следующему пути:
.github/workflows/build.yml

39. Вставьте в файл следующее содержимое:

name: Windows C Build (GCC/MSYS2)

on: [push, pull_request]

jobs:
  build:
    runs-on: windows-latest
    defaults:
      run:
        shell: msys2 {0}

    steps:
    - name: Check out repository code
      uses: actions/checkout@v4

    - name: Setup MSYS2 and GCC
      uses: msys2/setup-msys2@v2
      with:
        msystem: UCRT64
        update: true
        install: git mingw-w64-ucrt-x86_64-gcc make

    - name: Compile C code
      # The 'gcc' command is available in the MSYS2 shell
      run: gcc source/main.c -o hello.exe

    - name: Run the compiled program
      run: |
        ./hello.exe

    - name: Upload artifact
      uses: actions/upload-artifact@v4
      with:
        name: chip8.exe
        path: ./chip8.exe
        retention-days: 1

40. Зафиксируйте изменения и синхронизируйте с удаленным репозиторием.

41. Перейдите во вкладку Actions на GitHub и дождитесь успешного завершения процесса build.

42. Нажмите по первому в списке действию build.yml, пролистайте вниз до секции Artifacts. Скачайте и запустите полученный файл chip8.exe, чтобы убедиться в его работоспособности.

43. Создайте статью в TiddlyWiki с объяснением значения всех строк в файле build.yml Зафиксируйте проделанные изменения в локальном и удаленном репозиториях.

50. Вносите изменения в файл main.c. Фиксируйте и синхронизируйте изменения в GitHub после каждого из пронумерованных далее шагов.

51. Импортируйте библиотеку stdint.h - она определяет дополнительные типы данных, имеющих фиксированную (независящую от аппаратной платформы длину). Например, uint8_t, uint16_t.

52. Определите следующие константы на уровне макросов:
CHIP8_MEMORY_SIZE	4096
CHIP8_DISPLAY_WIDTH	64
CHIP8_DISPLAY_HEIGHT	32
V_REGISTER_COUNT	16
STACK_DEPTH		16
CHIP8_KEY_COUNT		16

53. Создайте статью в TiddlyWiki и объясните примерное значение этих параметров.

54. Определите структуру данных (typedef struct Chip8State) с любыми переменными типа uint8_t или uint16_t. Функционально мы заполним ее на следующих занятиях.


55. Создайте статью в TiddlyWiki с объяснением сущности и назначения структур данных.
