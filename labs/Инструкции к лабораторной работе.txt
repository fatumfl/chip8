# Знакомство с системой контроля версий git

1. Проверить в консоли установленную версию git:
git --version

2. Создать или перейти в папку, в которой будет располагаться наш репозиторий:
mkdir chip8
cd /chip8

3. Инициализировать новый репозиторий в целевой папке:
git init

4. Настроить параметры пользователя:
git config user.name "Имя.Фамилия"
git config user.email "sample@mail.com"
git config --local --list

5. Проверьте содержимое папки:
ls -a

6. Проверьте статус репозитория:
git status

7. Создайте следующую структуру папок и файлов в репозитории:

chip8/
	docs/
	roms/
	source/
	README.md

8. Добавьте все папки и файлы в список отслеживаемых:
git status
git add README.md		- добавит только указанный файл
git add .			- добавит все новые файлы в репозитории
git status

9. Зафиксируйте изменения в истории:
git commit -m "Comment"	- всегда используйте английский для комментариев с краткой сутью сделанных изменений
git status

10. Внесите изменения в файл README.md
	- На первой строке укажите имя проекта - CHIP-8 Emulator со стилем заголовка первого уровня
	- Укажите Имя, Фамилию, номер группы курсивом
	- Оставьте одну пустую строку и добавьте линию-разделитель для абзацев
	- Добавьте абзац с Lorem Ipsum

11. Зафиксируйте изменения:
git status
git add .
git status
git commit -m "Comment"

12. Просмотрите историю коммитов:
git log

13. Скачайте свежую версию TiddlyWiki и поместите в папку docs/ под именем index.html

14. Зафиксируйте изменения


# Работа с онлайн-репозиторием GitHub

15. Откройте приватный режим браузера
(Shift + CTRL + N - в Chromium браузерах)
(Shift + CTRL + P - в Firefox)

16. Зарегистрируйтесь или войдите на платформу GitHub

17. Создайте новый репозиторий со следующими параметрами:
	- Имя репозитория: chip8
	- Описание: Учебный эмулятор для систем на базе языка CHIP-8
	- Тип: Публичный
	- Галочка README: Снята
	- Шаблон .gitignore: None
	- Тип лицензии: Без лицензии

18. Скопируйте HTTP ссылку для клонирования репозитория

19. Свяжите локальный и удаленный репозитории:
git remote add origin <Ссылка на репозиторий>

20. Проверьте, что связь успешно установилась:
git remote -v

21. Отправка локальных данных в удаленный репозиторий GitHub:
git branch -M main
git push -u origin main

22. В дальнейшем, для синхронизации локального и удаленного репозитория, используйте порядок команд:
git add .
git commit -m "Comment"
git push

23. Изучите типы доступных лицензий и создайте статью в TiddlyWiki со сравнением особенностей и областей применения различных лицензий - "Лицензирование исходных кодов программного обеспечения" (задание на дом). Добавьте выбранный вами тип лицензии в репозиторий. Зафиксируйте проделанные изменения в локальном и удаленном репозиториях.
